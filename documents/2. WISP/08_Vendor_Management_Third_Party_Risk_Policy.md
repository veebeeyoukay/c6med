## 1. Purpose and Scope

### 1.1 Purpose
This Vendor Management and Third-Party Risk Policy establishes a framework for selecting, managing, and monitoring third-party vendors to ensure they meet C6Med LLC's security requirements, protect pharmaceutical client data, and support our business objectives while managing associated risks.

### 1.2 Scope
This policy applies to:
- All vendors, suppliers, and service providers
- Contractors and consultants with system/data access
- Cloud service providers and SaaS applications
- Professional services firms
- Any third party handling C6Med or client data
- Vendor selection, onboarding, monitoring, and termination

### 1.3 NIST CSF Alignment
This policy implements the following NIST CSF 2.0 controls:
- ID.SC-1: Cyber supply chain risk management processes identified
- ID.SC-2: Suppliers and partners identified and prioritized
- ID.SC-3: Contracts include cybersecurity requirements
- ID.SC-4: Suppliers assessed routinely
- ID.SC-5: Response and recovery planning with suppliers

## 2. Vendor Risk Management Framework

### 2.1 Risk-Based Approach
Vendors are managed according to their risk level, determined by:
- Access to C6Med systems or data
- Type and sensitivity of data handled
- Criticality to business operations
- Potential impact if compromised
- Regulatory/compliance implications

### 2.2 Vendor Categories

**Critical Vendors:**
- Core business platforms (Microsoft 365, Dropbox)
- Client data processors
- Financial services
- Telecommunications/ISPs
- Insurance providers

**High-Risk Vendors:**
- IT service providers
- Professional services with data access
- HR/Payroll services
- Marketing platforms with PII
- Contractors with system access

**Medium-Risk Vendors:**
- Training providers
- Office supplies/equipment
- Software licenses (non-critical)
- Consulting services (limited access)

**Low-Risk Vendors:**
- Vendors with no data access
- One-time service providers
- Public information only
- Standard commercial products

## 4. Vendor Lifecycle Management

### 4.1 Vendor Selection and Due Diligence

**Pre-Selection Requirements:**
1. Business justification documented
2. Alternative vendors considered
3. Risk assessment completed
4. Budget approval obtained
5. Security requirements defined

**Due Diligence Checklist:**

| Requirement | Critical | High | Medium | Low |
|------------|----------|------|--------|-----|
| Security questionnaire | Required | Required | Recommended | Optional |
| Financial stability review | Required | Required | Recommended | Optional |
| Reference checks | Required | Required | Optional | Optional |
| Insurance verification | Required | Required | Recommended | Optional |
| Compliance certifications | Required | Recommended | Optional | Optional |
| Security audit/SOC report | Required | Recommended | Optional | Not needed |
| Site visit (virtual/physical) | Recommended | Optional | Optional | Not needed |

### 4.2 Risk Assessment Process

**Assessment Components:**
1. **Data Security**
   - Encryption capabilities
   - Access controls
   - Data location/residency
   - Backup procedures
   - Incident response

2. **Compliance**
   - Regulatory compliance
   - Industry certifications
   - Privacy practices
   - Audit rights

3. **Operational**
   - Business continuity
   - Financial stability
   - Service reliability
   - Support quality

4. **Contractual**
   - Liability terms
   - Indemnification
   - Termination rights
   - SLA commitments

**Risk Scoring Matrix:**

| Factor | Weight | Score (1-5) | Weighted Score |
|--------|--------|-------------|----------------|
| Data sensitivity | 30% | | |
| Access level | 25% | | |
| Business criticality | 20% | | |
| Security maturity | 15% | | |
| Compliance | 10% | | |

### 4.3 Vendor Onboarding

**Onboarding Checklist:**
- [ ] Risk assessment completed

- [ ] NDA/confidentiality agreement signed

## 5. Security Requirements

### 5.1 Minimum Security Standards

**All Vendors with Data Access Must:**
- Maintain appropriate security policies
- Use encryption for sensitive data
- Implement access controls
- Provide breach notification
- Maintain incident response capability
- Carry appropriate insurance

**Additional for Critical/High-Risk:**
- Multi-factor authentication
- Regular security assessments
- Employee background checks
- Security awareness training
- Business continuity planning
- Annual security attestation

### 5.2 Cloud Service Requirements

**Required Features:**
- SOC 2 Type II or ISO 27001 certification
- Data encryption at rest and in transit
- Geographic data control
- Audit logging
- API security
- DDoS protection
- Regular backups
- Clear data ownership

**Preferred Features:**
- HIPAA compliance capability
- EU/US data residency options
- Advanced threat protection
- Zero-knowledge architecture
- Automated compliance reporting

### 5.3 Contractor/Consultant Requirements

**Before Granting Access:**
- Background check (where appropriate)
- NDA/confidentiality agreement
- Security awareness training
- Acceptable use acknowledgment
- Limited, role-based access
- Time-bound access
- Supervised initially

**Ongoing Requirements:**
- Activity monitoring
- Regular access reviews
- Immediate termination capability
- No local data storage
- Use of company-approved tools only

## 6. Specific Vendor Requirements

### 6.1 Critical Infrastructure Vendors

**Microsoft (Office 365, SharePoint, Teams):**
- Monitor service health dashboard
- Review security recommendations monthly
- Enable security defaults
- Configure conditional access
- Regular license optimization
- Maintain admin contacts

**Dropbox Business:**
- Enable team folders
- Configure sharing restrictions
- Monitor external collaborations
- Review activity logs
- Enforce 2FA
- Regular permission audits

**Internet/Telecom Providers:**
- SLA monitoring
- Redundancy where feasible
- Security incident reporting
- Regular service reviews
- Emergency contact maintenance

### 6.2 Professional Services

**Legal/Accounting Firms:**
- Professional liability insurance
- Confidentiality agreements
- Secure document exchange
- Data retention agreements
- Conflict of interest checks

**IT Services/Consultants:**
- Technical competency verification
- Security clearance (if needed)
- Supervised access initially
- Activity logging
- Knowledge transfer requirements

### 6.3 Client-Mandated Vendors

**When Clients Require Specific Vendors:**
- Document client requirement
- Perform modified risk assessment
- Note risk acceptance by client
- Implement compensating controls
- Monitor more frequently
- Document any concerns

## 7. Risk Management

### 7.1 Risk Assessment Triggers

**Conduct New Assessment When:**
- New vendor relationship
- Significant contract changes
- Security incident occurs
- Service expansion
- Ownership changes
- Regulatory changes
- Annual review cycle

### 7.2 Risk Treatment Options

**For Identified Risks:**
1. **Avoid**: Don't use vendor
2. **Mitigate**: Implement controls
3. **Transfer**: Insurance/contracts
4. **Accept**: Document acceptance

### 7.3 Concentration Risk

**Avoid Over-Reliance:**
- Identify single points of failure
- Maintain vendor alternatives
- Avoid vendor lock-in
- Regular market assessment
- Contingency planning

## 8. Incident Management

### 8.1 Vendor Incident Response

**When Vendor Incident Occurs:**
1. Immediate notification to Security Coordinator
2. Assess impact on C6Med
3. Activate incident response if needed
4. Coordinate with vendor
5. Client notification if affected
6. Document timeline and actions
7. Post-incident review
8. Contract/relationship review

### 8.2 Breach Notification Requirements

**Vendor Must Notify C6Med:**
- Within 24-48 hours of discovery
- Include preliminary assessment
- Provide affected data categories
- Describe containment actions
- Offer assistance/remediation
- Provide final report

### 8.3 Performance Issues

**Escalation Path:**
1. Operational contact
2. Account manager
3. Vendor management
4. Executive escalation
5. Legal action if necessary
