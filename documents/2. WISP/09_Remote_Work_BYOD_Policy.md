## 1. Purpose and Scope

### 1.1 Purpose
This Remote Work and Bring Your Own Device (BYOD) Policy establishes security requirements and guidelines for C6Med LLC's distributed workforce to safely work from home offices and use personal devices while protecting pharmaceutical client data and maintaining productivity.

### 1.2 Scope
This policy applies to:
- All C6Med employees working remotely
- All personal devices used for C6Med business
- Home office environments and alternative work locations
- Access to company and client data from any location
- All forms of remote connectivity

### 1.3 NIST CSF Alignment
This policy implements the following NIST CSF 2.0 controls:
- PR.AC-3: Remote access is managed
- PR.AC-4: Access permissions incorporate least privilege
- PR.DS-1: Data-at-rest is protected
- PR.MA-2: Remote maintenance is approved and logged
- DE.CM-7: Monitoring for unauthorized devices

## 2. Remote Work Authorization

### 2.1 Eligibility
All C6Med employees are authorized for remote work as the company operates on a fully distributed model. This includes:
- Full-time remote work from home offices
- Temporary remote work from alternate locations
- Travel-related remote work
- Emergency remote work situations

### 2.2 Remote Work Agreement
**Employees Must Acknowledge:**
- Understanding of remote work security requirements
- Responsibility for home office security
- Compliance with all company policies
- Incident reporting obligations
- Data protection requirements
- Equipment care responsibilities

### 2.3 Work Location Requirements
**Primary Home Office:**
- Dedicated workspace when feasible
- Secure, private environment
- Reliable internet connection
- Adequate lighting and ergonomics
- Lockable storage for sensitive materials

**Alternative Locations:**
- Business travel accommodations
- Co-working spaces
- Client offices
- Public spaces (with restrictions)

## 3. BYOD Program Requirements

### 3.1 Authorized Devices

**Approved Device Types:**
- Laptops/Desktops (Windows 10/11, macOS 12+)
- Tablets (iPad, Surface)
- Smartphones (iOS 14+, Android 11+)
- Other devices with CEO approval

**Minimum Device Requirements:**
- Supported, patched operating system
- 8GB RAM (recommended 16GB)
- Adequate storage for work files
- Camera and microphone for meetings
- Ability to run required software

### 3.2 Device Security Requirements

**Mandatory Security Controls:**

| Control | Requirement | Verification |
|---------|------------|--------------|
| Device encryption | Full disk encryption enabled | Screenshot proof |
| Screen lock | Password/PIN/biometric, 5-min timeout | Settings screenshot |
| Operating system | Current version, auto-updates enabled | Version check |
| Antivirus | Real-time protection active | Status screenshot |
| Firewall | Enabled with default settings minimum | Configuration check |
| Remote wipe | Capability enabled for company data | Enrollment confirmation |

### 3.3 BYOD Enrollment Process

1. **Device Assessment**
   - Complete device security checklist
   - Verify minimum requirements met
   - Document device information

2. **Security Configuration**
   - Enable required security features
   - Install required security software
   - Configure cloud storage access

3. **Agreement Execution**
   - Sign BYOD agreement
   - Acknowledge responsibilities
   - Accept monitoring terms

4. **Verification**
   - Submit security screenshots
   - Security Coordinator approval
   - Add to device inventory

### 3.4 Employee Responsibilities

**Device Owner Must:**
- Maintain device security
- Apply updates promptly (within 7 days)
- Report lost/stolen devices immediately
- Backup work data to approved cloud storage
- Separate personal and work data
- Allow security compliance checks
- Remove company data upon termination

### 3.5 Company Responsibilities

**C6Med Will:**
- Provide clear BYOD requirements
- Offer security configuration guidance
- Reimburse portion of costs (if applicable)
- Provide technical support for work applications
- Respect personal privacy on devices
- Not access personal data

## 4. Remote Access Security

### 4.1 Network Security Requirements

**Home Network Security:**
- WPA2/WPA3 encryption required
- Strong WiFi password (12+ characters)
- Default router password changed
- Guest network for IoT devices
- Router firmware updated
- Network name (SSID) not identifying

**Public Network Usage:**
- Avoid public WiFi for sensitive work
- Use mobile hotspot when possible
- VPN required on public networks (when implemented)
- No sensitive data access on public WiFi
- Verify network legitimacy

### 4.2 Authentication Requirements

**Remote Access Must Use:**
- Multi-Factor Authentication (MFA) for all cloud services
- Strong, unique passwords
- Password manager recommended
- Biometric authentication where available
- Session timeouts enforced

### 4.3 Cloud Service Access

**Approved Services:**
- Microsoft Office 365
- SharePoint Online
- Dropbox Business
- Microsoft Teams
- Other CEO-approved services

**Prohibited for Work:**
- Personal cloud storage (Google Drive, iCloud)
- Personal email accounts
- Unsanctioned file sharing services
- Social media for work files

## 5. Data Protection Requirements

### 5.1 Data Storage

**On Personal Devices:**
- RESTRICTED data: Prohibited on local storage
- CONFIDENTIAL data: Encrypted containers only
- INTERNAL data: Standard device encryption
- PUBLIC data: No restrictions

**Cloud Storage Requirements:**
- Use only approved cloud platforms
- Enable versioning and recovery features
- Do not sync sensitive data locally
- Regular cleanup of cached files

### 5.2 Data Handling

**Remote Workers Must:**
- Follow data classification policy
- Not store client data locally
- Use secure deletion for temporary files
- Clear browser caches regularly
- Not use personal email for work
- Encrypt sensitive email attachments

### 5.3 Printing and Physical Documents

**Home Printing Guidelines:**
- Minimize printing of sensitive documents
- Secure printed materials immediately
- Shred sensitive documents
- Do not leave documents visible during video calls
- Store client materials in locked location

## 6. Communication and Collaboration

### 6.1 Video Conferencing

**Security Requirements:**
- Use company-approved platforms
- Enable waiting rooms
- Use meeting passwords for sensitive topics
- Do not record without consent
- Verify participants before discussing confidential matters
- Use virtual backgrounds to hide location

**Environmental Considerations:**
- Check background for sensitive information
- Use headphones in shared spaces
- Mute when not speaking
- Ensure privacy from household members
- Test connection before client meetings

### 6.2 Messaging and Email

**Secure Communications:**
- Use Teams for internal communication
- Email encryption for sensitive data
- Verify recipient addresses
- Do not forward work emails to personal accounts
- Report suspicious messages immediately

### 6.3 File Sharing

**Approved Methods:**
- SharePoint document libraries
- Dropbox shared folders
- Teams file sharing
- Email (with encryption for sensitive)

**Prohibited Methods:**
- Personal file sharing services
- USB drives to personal computers
- Social media or messaging apps
- Unsecured email attachments

## 7. Physical Security

### 7.1 Home Office Security

**Physical Controls:**
- Lock devices when stepping away
- Position screens away from windows
- Secure devices at end of workday
- Lock sensitive documents
- Control visitor access during work hours
- Use privacy screens in shared spaces

### 7.2 Travel Security

**When Traveling:**
- Keep devices with you or in hotel safe
- Use privacy screens in public
- Avoid sensitive calls in public areas
- Be aware of shoulder surfing
- Use RFID-blocking cases
- Wipe devices before international travel (if required)

### 7.3 Device Disposal

**Before Disposing Personal Device:**
- Transfer all work data to approved storage
- Obtain data removal certification
- Factory reset device
- Physically destroy storage if contained sensitive data
- Document disposal method

## 8. Technical Support

### 8.1 Support Boundaries

**C6Med Provides Support For:**
- Work application issues

**Employee Responsible For:**

- Disciplinary action

## 10. Incident Response

### 10.1 Lost or Stolen Devices

**Immediate Actions:**
1. Report to Security Coordinator immediately
2. Attempt remote lock/wipe if possible
3. Change all passwords
4. File police report if stolen
5. Contact insurance if applicable
6. Document device contents

### 10.2 Security Incidents

**If Compromise Suspected:**
1. Disconnect device from network
2. Contact Security Coordinator
3. Do not attempt to fix yourself
4. Preserve evidence
5. Document observations
6. Await instructions

### 10.3 Data Breach

**If Data Exposed:**
1. Stop the exposure immediately
2. Report to Security Coordinator
3. Document what was exposed
4. Identify who may be affected
5. Preserve all evidence
6. Cooperate with investigation
